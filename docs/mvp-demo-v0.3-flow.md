## Katris Travel MVP Demo v0.3 Flow

### Demo 输入

```txt
Departure: LAX
Destination: JFK
Travel date: 未来 30 天后
Travelers: 1
Budget: 1000 USD
Notes: 不要红眼航班
```

### Demo 操作路径

1. 生成计划。
2. 查看 `推荐执行方案`。
3. 查看航班决策卡。
4. 选择一个航班。
5. 查看推荐酒店是否进入预订清单。
6. 查看每日路线锚点。
7. 查看旅行消耗力分析。
8. 查看预订清单。
9. 点击“我已购买”。
10. 填写确认号 `ABC123`。
11. 点击“生成最终旅行执行单”。
12. 在浏览器打印窗口中保存为 PDF。
13. 开启旅行中助手。
14. 切换“只看重要提醒 / 标准提醒”。
15. 忽略一条提醒。
16. 修改预算。
17. 确认旧推荐进入 stale 状态。

### 预期结果

- 页面不崩溃。
- 没有明显 console error。
- 推荐路径不等于已确认。
- 航班选择状态正确。
- 酒店推荐进入清单但不是 `confirmed`。
- `tripEffort` 显示正常。
- `bookingChecklist` 显示正常。
- 确认号进入执行单。
- PDF 打印窗口可打开。
- 旅行中助手能显示提醒。
- 右侧助手摘要同步更新。
- 修改预算后旧推荐标记为 stale。

### 演示说明

- `推荐执行方案` 的语义是“系统推荐，尚未完成购买确认”。
- 只有用户手动选择航班、点击“我已购买”、保存确认号后，相关项目才会进入更高确认状态。
- stale 提示出现后，应重新点击“生成计划”，不要继续基于旧推荐完成购买。
